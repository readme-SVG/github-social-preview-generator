# Security Policy

## Supported Versions

The project is currently maintained on the default branch. Security updates are provided for the latest code in the repository.

| Version | Supported |
| --- | --- |
| Latest (`main`) | ✅ |
| Older releases / snapshots | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, report it **privately** by email to:

- **security@your-project-domain.example**

Please include as much detail as possible:

- A clear description of the issue and impact
- Reproduction steps or proof of concept
- Affected files, versions, or commit hashes
- Any suggested mitigations

### Do not report security issues publicly

Security vulnerabilities **must not** be reported through public GitHub Issues, Discussions, pull requests, or any other public channel.

Public disclosure before a fix is available can put users at risk. Use only the private reporting channel above for security findings.

## API Keys and Data Privacy

This project may use third-party API keys to provide its features. All processing is performed locally or strictly client-side in the user’s own environment/browser.

This project does **not** collect, log, transmit, or store:

- User API keys
- Personal data
- Request/response payloads on external project-managed servers

Users are responsible for protecting their own API keys and configuring their local/runtime environment securely.
